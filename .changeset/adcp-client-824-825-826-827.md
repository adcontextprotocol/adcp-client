Dogfooding follow-ups from reference training agent (adcp#2889). Four
bundled fixes across the server + testing surfaces:

- **`bridgeFromSessionStore(loadSession, selectSeededProducts, defaults?)`**
  (adcp-client#824) — new helper for sellers whose seed store is
  session-scoped (one Map per tenant / brand.domain / account_id, loaded
  per request). The existing `bridgeFromTestControllerStore` takes a
  single Map at construction time and doesn't compose with per-request
  session loading; the new helper takes two callbacks instead.

- **`mcpAcceptHeaderMiddleware` now rewrites `req.rawHeaders`**
  (adcp-client#825) — the MCP SDK's `StreamableHTTPServerTransport`
  rebuilds its Fetch `Headers` from `req.rawHeaders` via
  `@hono/node-server`, ignoring `req.headers`. Patching only the parsed
  map was a silent no-op for the transport the middleware's name
  implies. Both surfaces now move in lockstep (case-insensitive on the
  rawHeaders name, all duplicate entries rewritten, no phantom header
  added when `rawHeaders` lacks an Accept entry).

- **`adcpError()` consults a per-code inside-`adcp_error` allowlist**
  (adcp-client#826) — new `ADCP_ERROR_FIELD_ALLOWLIST` map in
  `@adcp/client/server` (parallel to the existing
  `ERROR_ENVELOPE_FIELD_ALLOWLIST` for siblings). The builder filters
  its output against the allowlist for the given code. `IDEMPOTENCY_CONFLICT`
  is the canonical strict case: `recovery`, `field`, `suggestion`, and
  `details` drop from the wire shape so the envelope can't become a
  stolen-key read oracle. Codes without an entry pass through unchanged.
  `AdcpErrorPayload.recovery` is now optional to reflect the filtered
  wire shape. Legacy `CONFLICT_ADCP_ERROR_ALLOWLIST` stays exported as
  an alias for the `IDEMPOTENCY_CONFLICT` entry.

- **`createDefaultTestControllerStore` docstring clarifies fit**
  (adcp-client#827) — calls out that the helper's "collapse 300 lines
  to 10" claim applies to sellers whose session state IS
  `DefaultSessionShape`. Sellers whose production tools read from richer
  domain types (typed `MediaBuyState`, `CreativeState`,
  `GovernancePlanState`) should build a `TestControllerStore` against
  `enforceMapCap` + `createSeedFixtureCache` instead — the default
  handlers write into a parallel bag of Maps those tools don't read,
  causing silent drift between seed and retrieval.

Who is affected: server-side agent authors using `@adcp/client/server`
or `@adcp/client/testing` to stand up a conformance-capable agent.
Upgrade path: no action required for existing callers —
`bridgeFromTestControllerStore` and the legacy `CONFLICT_ADCP_ERROR_ALLOWLIST`
export are unchanged. Agents that emit `recovery` on an
`IDEMPOTENCY_CONFLICT` envelope via their own (non-`adcpError()`) code
path will now be flagged by the `idempotency.conflict_no_payload_leak`
default invariant — the fix is to let the SDK builder emit the envelope
(recommended) or drop `recovery` from hand-rolled conflict responses.
