feat(storyboard): runner-output-contract v2.0.0 — forward-compat default, capture/substitution synthesized checks, `upstream_traffic` (adcp-client#1253, spec adcp#3816)

Implements the runner-side of the unified anti-façade + cascade-attribution contract landing in the spec. Three additive features, all backwards-compatible for storyboards that don't opt in.

**Forward-compat default for unknown check kinds.** When the runner encounters an authored `check` value it does not implement (e.g. a storyboard declares a check added in a later spec minor version), it now grades the validation as `passed: true, not_applicable: true` with a `note` describing the coverage gap, instead of failing the step with `Unknown validation check: …`. New per-run counter `validations_not_applicable` surfaces the count on `StoryboardResult`, `StoryboardPassResult`, and the compliance `summary` so dashboards can distinguish "runner is older than the storyboard" from clean passes. Independently valuable — unblocks any future additive check-type extension from breaking older runners.

**`capture_path_not_resolvable` synthesized checks.** When a step's `context_outputs.path` resolves to absent, `null`, or `""`, the runner now synthesizes a failed `validation_result` on the *capturing* step (check: `capture_path_not_resolvable`, `expected`: declared path, `actual`: resolved value, `json_pointer`: RFC 6901 form of the path). Previously the runner silently skipped the capture and the failure surfaced — confusingly — on the downstream consumer. Fixes the cascade-attribution gap from adcp#3796. `applyContextOutputsWithProvenance` now also returns the failures on `ContextWriteResult.failures` for callers using the lower-level surface.

**`unresolved_substitution` synthesized checks.** When a downstream step is skipped because a `$context.<key>` token can't resolve, the skipped step's `validations[]` now carries an `unresolved_substitution` validation result per missing token (`expected`: the token string; `actual` / `json_pointer` / `request` / `response` all `null` because no request was sent). `skip_result.reason` remains `prerequisite_failed`. Lets dashboards attribute the cascade origin without parsing skip messages.

**`upstream_traffic` check — load-bearing anti-façade contract.** New authored check that asserts side-effects against the adopter's `comply_test_controller`'s `query_upstream_traffic` scenario. Adopters who advertise the scenario in `list_scenarios` opt in; adopters who don't grade `not_applicable`. Adopters who do but observed zero recorded calls in the assertion window grade `failed` (the façade signal). Supports `min_count`, `endpoint_pattern` glob, `payload_must_contain` (path + match: `present` / `equals` / `contains_any`), and `buyer_identifier_echo: true` shorthand (asserts that any buyer-identifier-shaped value from the storyboard's `sample_request` appears in at least one matching call's payload — placeholder constants don't match the storyboard's vectors).

**Surface details.** `ValidationResult` gains optional `not_applicable` and `note` fields. `StoryboardValidationCheck` gains `upstream_traffic`. `StoryboardValidation` gains `min_count` / `endpoint_pattern` / `payload_must_contain` / `buyer_identifier_echo` / `since` for upstream_traffic. `ControllerScenario` gains `query_upstream_traffic`. New helper `queryUpstreamTraffic` and types `RecordedCall` / `UpstreamTrafficSuccess` / `UpstreamTrafficQueryParams` exported from `testing/test-controller`. `UpstreamTrafficPayloadMatch` exported from `testing/storyboard`. `ValidationContext` gains `upstreamTraffic` and `storyboardStep` fields populated by the runner's pre-fetch (the synchronous validator gets prebuilt data — no async dispatcher).
