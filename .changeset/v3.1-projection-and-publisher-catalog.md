feat(v2): v1↔v2 Product projection layer + publisher-catalog helpers + format_schema fetcher

Opt-in toolkit for adopters who want the AdCP 3.1 V2 mental model (`Product.format_options[]`) while talking to v1 or v2 sellers. All exports are additive — existing 7.x code reading `Product.format_ids[]` keeps working without changes. Lands as one piece because the projection layer, publisher-catalog helpers, and format_schema fetcher are mutually dependent for end-to-end V2 buying.

### What's new

- **`@adcp/sdk/v2/projection`** — bidirectional v1↔v2 Product projection with the spec's 4-step resolution order (catalog `canonical` annotation → registry glob → structural → fail-closed). Five diagnostic codes: `FORMAT_PROJECTION_FAILED`, `FORMAT_DECLARATION_V1_AMBIGUOUS`, `FORMAT_DECLARATION_V1_NOT_APPLICABLE`, `CANONICAL_NOT_V1_TRANSLATABLE`, `FORMAT_DECLARATION_V1_LOSSY_MULTI_SIZE`.

  ```ts
  import { withFormatOptions } from '@adcp/sdk/v2/projection';

  const v1Response = await agent.getProducts({ brief: '...' });
  const { response, diagnostics } = withFormatOptions(v1Response);
  // response.products[i].format_options[] now carries the V2 declaration
  // alongside the original format_ids[]; diagnostics surface what didn't project.
  ```

- **`@adcp/sdk/v2/publisher-catalog`** — extract / scope / capability-resolve helpers on top of the existing `validateAdAgents()` discovery layer. Consumes `adagents.json#/formats[]` (AdCP 3.1 publisher-published format catalog) and the AAO community-mirror catalogs for unadopted platforms.

  ```ts
  import { validateAdAgents } from '@adcp/sdk/discovery';
  import {
    extractPublisherFormats,
    scopePublisherFormats,
    resolveCapabilityId,
  } from '@adcp/sdk/v2/publisher-catalog';

  const { adagents } = await validateAdAgents('nytimes.com');
  const homepageFormats = scopePublisherFormats(extractPublisherFormats(adagents), {
    propertyId: 'homepage',
  });
  const placementFormat = resolveCapabilityId(homepageFormats, 'nytimes_homepage_takeover_premium');
  ```

- **`@adcp/sdk/v2/format-schema`** — `fetchFormatSchema({ uri, digest })` implements the spec's normative fetch contract for `ProductFormatDeclaration.format_schema`: HTTPS-only via `ssrfSafeFetch`, 1 MiB body cap, 5 s timeout, SHA-256 digest verification, manual redirect handling, immutable `uri@digest` caching. Reuses the existing SSRF guards + DNS pinning from the discovery layer.

  ```ts
  import { fetchFormatSchema, FormatSchemaFetchError } from '@adcp/sdk/v2/format-schema';

  const { schema, fromCache } = await fetchFormatSchema({
    uri: 'https://nytimes.example/format-schemas/takeover.json',
    digest: 'sha256:...',
  });
  ```

- **`AdAgentsJson.formats?`** — extension on the discovery types so `validateAdAgents()` returns the publisher-published format catalog where present. Optional and backward-compatible with 3.0.x adagents.json files (no field present → no change in behavior).

- **`v1_format_ref` is now `V1FormatId[]`** in projection types — mirrors the 3.1-beta normative schema (`minItems: 1`). Single-ref is `[{...}]`; multi-size carries one entry per size with the projection layer auto-fanning when the seller asserts < `sizes.length` refs.

### Why opt-in (not auto-wired into `AgentClient`)

7.10 keeps `format_ids` on the public Product type so the 7.x adoption path is purely additive — no silent payload mutation, no TypeScript inference skew for callers who haven't moved. The 8.0 release narrows the public surface to the V2 mental model and moves `withFormatOptions` onto the default response path. The spec's v1 deprecation floor (2027-Q4) is the trigger for that move.

### Tests

80 new tests across the projection layer, publisher catalog, format-schema fetcher, V1↔V2 round-trip matrix (every AAO catalog entry round-trips to itself), and cross-version smoke harness. The matrix proves a buyer on V2 sees the same product regardless of which side of the wire the SDK is bridging.

### Out of scope (deferred)

- Auto-projection inside `AgentClient.getProducts()` — buyers call `withFormatOptions` explicitly at 7.10; 8.0 moves it onto the default path.
- Manifest-layer `pixel_tracker` v1↔v2 translation (with the `sync_creatives` wiring).
- `format_schema` `$ref` sandboxing (depth/count bounds) — Ajv loader-hook concern at validation time, not fetch time.
- Schema-compile DoS bounds (re2 pattern engine, allOf expansion) — applied at the Ajv compile step in the manifest validator.
