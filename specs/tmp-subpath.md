# Spec: `@adcp/client/tmp` Subpath Export

## Goal

Add a `/tmp` subpath export to `@adcp/client` that provides types and pure
functions for building TMP requests, processing responses, and joining context
match with identity match results. This enables Prebid.js (and other platforms)
to integrate TMP without writing protocol logic.

## Design constraints

- **Zero dependencies.** Types and pure functions only.
- **Tree-shakeable.** ESM named exports. Import only what you need.
- **No HTTP client.** Consumers (Prebid, custom integrations) bring their own
  transport. The SDK does not make network calls.
- **Bundle target: <3KB gzipped** for types + all functions.

## Types

TMP schema types are already generated by the `json-schema-to-typescript`
pipeline in `scripts/generate-types.ts` and land in `core.generated.ts`.
The TMP schemas (`context-match-request`, `context-match-response`, etc.) will
be included automatically when the next AdCP schema version is published.

The `/tmp` subpath re-exports the generated types with cleaner names and adds
SDK-specific types not derived from schemas:

```typescript
// Re-exported from generated types
export type {
  ContextMatchRequest,
  ContextMatchResponse,
  IdentityMatchRequest,
  IdentityMatchResponse,
  Offer,
  OfferPrice,
  PackageEligibility,
  AvailablePackage,
} from '../types/core.generated';

// SDK-specific types
export interface Artifact {
  type: 'url' | 'url_hash' | 'eidr' | 'gracenote' | 'isrc' | 'gtin' | 'rss_guid' | 'isbn' | 'custom';
  value: string;
}

export interface ContextSignals {
  topics?: string[];
  taxonomy_source?: 'iab';
  taxonomy_id?: number;
  sentiment?: 'positive' | 'negative' | 'neutral' | 'mixed';
  keywords?: string[];
  language?: string;
  content_policies?: string[];
  summary?: string;
  embedding?: string;
  embedding_model?: string;
  embedding_dims?: number;
}

export interface Consent {
  gdpr?: boolean;
  tcf_consent?: string;
  gpp?: string;
  us_privacy?: string;
}

export interface JoinResult {
  activated: ActivatedOffer[];
  skipped: SkippedOffer[];
  signals?: ContextMatchResponse['signals'];
}

export interface ActivatedOffer {
  offer: Offer;
  eligibility: PackageEligibility;
}

export interface SkippedOffer {
  offer: Offer;
  reason: 'not_eligible' | 'no_eligibility_data';
}
```

## Functions

### `buildContextMatchRequest`

```typescript
export function buildContextMatchRequest(params: {
  propertyRid: string;
  propertyType: string;
  placementId: string;
  artifacts?: Artifact[];
  contextSignals?: ContextSignals;
  geo?: { country?: string; region?: string; metro?: { system: string; value: string } };
  packageIds?: string[];
  propertyId?: string;
}): ContextMatchRequest;
```

- Generates `request_id` via `crypto.randomUUID()`
- Sets `protocol_version: "1.0"`
- Validates required fields (throws on missing `propertyRid`, `propertyType`, `placementId`)
- Returns a plain object ready for `JSON.stringify()`

### `buildIdentityMatchRequest`

```typescript
export function buildIdentityMatchRequest(params: {
  userToken: string;
  uidType: string;
  packageIds: string[];
  consent?: Consent;
}): IdentityMatchRequest;
```

- Generates `request_id` via `crypto.randomUUID()`
- Sets `protocol_version: "1.0"`
- Validates required fields (throws on missing `userToken`, empty `packageIds`)

### `joinResults`

```typescript
export function joinResults(
  context: ContextMatchResponse,
  identity: IdentityMatchResponse,
): JoinResult;
```

Intersects context match offers with identity match eligibility:
1. Build a Map of `package_id → PackageEligibility` from identity response
2. For each offer in context response:
   - If eligibility exists and `eligible === true` → `activated`
   - If eligibility exists and `eligible === false` → `skipped` (reason: `not_eligible`)
   - If no eligibility data → `skipped` (reason: `no_eligibility_data`)
3. Sort activated by `intent_score` descending (offers without scores sort last)
4. Preserve `context.signals` on the result

### `toTargetingKVs`

```typescript
export function toTargetingKVs(result: JoinResult): Record<string, string[]>;
```

Flattens a JoinResult to GAM-compatible key-values:
- `adcp_pkg` → activated package IDs (in intent_score order)
- `adcp_seg` → segment strings from `result.signals.segments`
- Additional keys from `result.signals.targeting_kvs`, grouped by key

## File structure

```
src/lib/tmp/
  index.ts                  # re-exports everything
  types.ts                  # SDK-specific types (JoinResult, Artifact, etc.)
  build-context-match.ts    # buildContextMatchRequest
  build-identity-match.ts   # buildIdentityMatchRequest
  join-results.ts           # joinResults
  to-targeting-kvs.ts       # toTargetingKVs
```

Add to existing `exports` and `typesVersions`:

  "exports": {

## Usage: Prebid.js adapter

```typescript
import {
  buildContextMatchRequest,
  buildIdentityMatchRequest,
  joinResults,
  toTargetingKVs,
} from '@adcp/client/tmp';

// Build context match request from ad unit config
const contextReq = buildContextMatchRequest({
  propertyRid: config.propertyRid,
  propertyType: 'website',
  placementId: adUnit.tmp.placementId,
  artifacts: [{ type: 'url', value: window.location.href }],
});

// Send via Prebid's AJAX (SDK does not make HTTP calls)
const contextRes = await prebidAjax(routerUrl + '/context', contextReq);

// After temporal delay...
const identityReq = buildIdentityMatchRequest({
  userToken: uid2Token,
  uidType: 'uid2',
  packageIds: allActivePackageIds,
});
const identityRes = await prebidAjax(routerUrl + '/identity', identityReq);

// Join and set targeting
const result = joinResults(contextRes, identityRes);
const kvs = toTargetingKVs(result);
setGAMTargeting(adUnit.code, kvs);
```

## Usage: standalone (no Prebid)

```typescript
import { buildContextMatchRequest } from '@adcp/client/tmp';

const req = buildContextMatchRequest({
  propertyRid: '01916f3a-9c4e-7000-8000-000000000010',
  propertyType: 'ai_assistant',
  placementId: 'chat-inline',
  contextSignals: { summary: 'User exploring Italian cookware options', topics: ['632'] },
});

const res = await fetch(routerUrl + '/context', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(req),
});
```
